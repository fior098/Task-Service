# Task Service - Решение тестового задания

## Описание реализованной функциональности

Реализована система периодических задач для медицинского трекера задач. Решение позволяет врачам создавать повторяющиеся задачи с различными типами периодичности.

## Принятые технические решения

### 1. Типы периодичности

Реализованы все требуемые типы периодических задач:

- **Ежедневные** (`daily`) - задача повторяется каждый N-й день
  - Параметр: `recurrence_interval` (минимум 1)
  - Пример: каждый день, каждые 3 дня, каждую неделю (7 дней)

- **Ежемесячные** (`monthly`) - задача на определенное число каждого месяца
  - Параметр: `recurrence_day_of_month` (от 1 до 30)
  - Ограничение до 30 числа для избежания проблем с февралем
  - Пример: 5-го числа каждого месяца, 15-го числа каждого месяца

- **Конкретные даты** (`specific_dates`) - задачи только на указанные даты
  - Параметр: `recurrence_specific_dates` (JSON массив дат)
  - Формат: `["2024-03-15", "2024-06-20", "2024-12-31"]`

- **Четные дни** (`even_days`) - задачи только на четные числа месяца
  - Автоматически: 2, 4, 6, 8... 30 число

- **Нечетные дни** (`odd_days`) - задачи только на нечетные числа месяца
  - Автоматически: 1, 3, 5, 7... 31 число

### 2. Архитектурные решения

#### Clean Architecture
Проект следует принципам чистой архитектуры:
- `domain` - бизнес-логика и сущности
- `usecase` - сценарии использования
- `repository` - работа с данными
- `transport` - HTTP handlers
- `infrastructure` - внешние зависимости (БД)

#### Разделение ответственности
- **Repository** - работа с базой данных
- **Service** - бизнес-логика создания/обновления задач
- **Scheduler** - логика вычисления и создания повторяющихся задач
- **Handler** - HTTP обработчики

### 3. Схема базы данных

Добавлены поля для поддержки периодичности:

```sql
recurrence_type VARCHAR(50)           -- тип периодичности
recurrence_interval INTEGER           -- интервал для daily
recurrence_day_of_month INTEGER       -- число месяца для monthly
recurrence_parity VARCHAR(10)         -- для четных/нечетных (deprecated)
recurrence_specific_dates TEXT        -- JSON массив дат
recurrence_end_date TIMESTAMP         -- дата окончания повторений
parent_task_id INTEGER                -- ссылка на родительскую задачу
